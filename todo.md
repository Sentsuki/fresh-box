# TODO List

## 性能与架构优化

- [ ] **优化大流量/多连接场景下的性能（连接速率计算）**
  - **需求描述**：当应用处理大量并发连接（如 P2P、BT 下载）时，前端进行速率计算和高频排序会导致 UI 帧率下降。需要将连接速率、活动连接总数等数据处理逻辑移出前端渲染层，由数据源直接提供计算好的最终结果，以减轻前端主线程负担，保证 UI 交互顺畅。
  
- [ ] **支持极致省内存的“销毁窗口”模式**
  - **需求描述**：为满足低配置机器或对内存敏感的用户需求，在设置中增加“销毁窗口”的选项。开启后，点击关闭按钮将彻底释放前端页面占用的全部内存，仅保留系统托盘图标常驻后台；当用户再次从托盘呼出应用时，重新创建并加载窗口。实现极致省内存与快速打开之间的平衡。
